## google-gemini/gemini-cli

直近7日間で v0.45.0（2026-06-03）、v0.45.1（2026-06-04）、v0.45.2（2026-06-05）がリリースされました。本体となる機能リリースは **v0.45.0** で、v0.45.1 / v0.45.2 はパッチ（cherry-pick）です。

### v0.45.0 の主な変更点

- **コンテキスト処理の簡素化**: コンテキスト周りの実装簡素化作業が完了（#27345）。
- **自動ルーティングのデフォルト更新**: モデル自動ルーティングの既定値を見直し（#27071）。ルーティング分類器をバイパスして orphaned function response エラーを防止する修正も追加（#27389）。
- **A2A の使用状況メタデータ公開**: Agent-to-Agent 連携で usage metadata を公開（#27288）。
- **セッション再開の改善**: 再開時に内部セッションコンテキストを履歴からフィルタリング（#27391）。
- **ターミナル/PTY 安定性向上**: Termux での再起動・リサイズ再マウントのループ防止（#27110）、PTY リサイズ時の EBADF エラー抑制（#27461）。
- **MCP のセキュリティ修正**: `mcp list` におけるブラックリスト回避を防止（#27377）。
- **Vim モード修正**: マッピングされていない vim normal キーを無視（#27102）。

### v0.45.1 / v0.45.2（パッチ）

- v0.45.1: 665228e の cherry-pick によるパッチ。
- v0.45.2: f40498d の cherry-pick によるパッチ。

### 開発者への影響

自動ルーティングのデフォルト変更により、モデル選択の挙動が変わる可能性があります。意図したモデルが選ばれない場合は明示的なモデル指定を検討してください。セッション再開（resume）周りの修正で、中断後の会話継続がより安定します。

## anthropics/claude-code

CHANGELOG.md の最終更新は **2026-06-06**（直近7日間）。最新の機能追加を含むエントリは **2.1.166**、最新版は **2.1.168** です（2.1.165/2.1.167/2.1.168 は "Bug fixes and reliability improvements"）。

### 2.1.166 の主な変更点

- **フォールバックモデル設定 `fallbackModel`**: プライマリモデルが過負荷・利用不可のときに順番に試す最大3つのフォールバックモデルを設定可能に。`--fallback-model` がインタラクティブセッションにも適用されるようになりました。
  - 設定例（settings.json）:
    ```json
    {
      "fallbackModel": ["claude-opus-4-8", "claude-sonnet-4-6", "claude-haiku-4-5"]
    }
    ```
  - API が想定外の non-retryable エラーを返した場合、フォールバックモデルで1回リトライするようになりました（認証・レート制限・リクエストサイズ・トランスポートエラーは即時表面化）。
- **deny ルールでの glob パターン対応**: ツール名位置で glob を使用可能に（`"*"` ですべてのツールを拒否）。allow ルールは非 MCP の glob を拒否し、deny ルールの未知ツール名は起動時に警告。
- **クロスセッションメッセージングのハードニング**: 他の Claude セッションから `SendMessage` 経由でリレーされたメッセージはユーザー権限を持たなくなり、受信側はリレーされた権限要求を拒否、auto モードではブロック。
- **thinking 無効化の改善**: `MAX_THINKING_TOKENS=0`、`--thinking disabled`、モデル別 thinking トグルが、Claude API でデフォルト thinking するモデルでも thinking を無効化（サードパーティプロバイダーは変更なし）。
- **`claude update`**: ダウンロード前に対象バージョンを表示するように。
- **`claude agents`**: 一覧で URL を入力すると、最初のプロンプトにその URL を含むセッションに絞り込み。

### 2.1.163 の主な変更点（参考: 直近の機能追加）

- **バージョン強制管理設定**: `requiredMinimumVersion` / `requiredMaximumVersion` を追加。許容範囲外だと起動を拒否し承認済みバージョンへ誘導。
- **`/plugin list` コマンド**: インストール済みプラグイン一覧（`--enabled` / `--disabled` フィルタ対応）。
- **Hooks 拡張**: Stop / SubagentStop フックが `hookSpecificOutput.additionalContext` を返してターンを継続可能に。
- **Skills**: コマンド本文で数字の前にリテラルの `$` を含めるための `\$` エスケープ構文を追加。

### 開発者への影響

`fallbackModel` により、モデル過負荷時の中断が減り長時間セッションの信頼性が向上します。deny ルールの glob 対応とクロスセッションメッセージングのハードニングは、チーム/エンタープライズ環境でのセキュリティ運用に有効です。

- リポジトリ: https://github.com/anthropics/claude-code
- CHANGELOG: https://github.com/anthropics/claude-code/blob/main/CHANGELOG.md

## microsoft/vscode

最新リリースは **1.123.0（2026-06-05）**（直近7日間）。本ダイジェストでは AI / GitHub Copilot 関連の変更のみを対象とします。

### AI / Copilot 関連の主な変更点

- **セッション同期と Chronicle**: チャットセッションが GitHub アカウントへ自動同期され、マシン横断で検索可能な履歴に。`/chronicle` コマンドで過去セッションへの自然言語クエリ、スタンドアップレポート生成、生産性インサイトを取得可能。会話・変更ファイル・リポジトリコンテキスト・参照した PR/Issue をキャプチャ。
- **Agents Window（プレビュー）**: 複数のエージェントセッションを並べて比較・レビュー可能。セッションのピン留め、最大化表示、Alt+クリックでのセッション起動に対応。アクティブセッションに応じてターミナル・ファイル・変更ビューが更新。
- **Research Agent（プレビュー）**: Copilot CLI のローカルセッションで利用可能（Insiders のみ）。`/research` コマンドで、コードベース・GitHub リポジトリ・Web ソースを統合し、引用付きの Markdown レポートを生成（読み取り専用アクセス）。
- **サンドボックスの改善**: ネットワーク依存のターミナルコマンドは、必要に応じて無制限アクセスで自動リトライし、その後サンドボックス外実行へフォールバック。
- **コンテキストウィンドウの拡張**: Anthropic / OpenAI の対応モデルで100万トークンのコンテキストウィンドウをサポート（Claude Opus 4.7、GPT-5.5 で動作）。
- **統合ブラウザのスクリーンショット連携**: 「Add Area Screenshot to Chat」（矩形選択キャプチャ）、「Add Full Page Screenshot to Chat（実験的）」（ビューポート外も含むページ全体キャプチャ）。

- リポジトリ: https://github.com/microsoft/vscode
- リリース: https://github.com/microsoft/vscode/releases/tag/1.123.0

### 0.137.0 の主な変更点

#### New Features

- **TUI コントロールの拡張**: F13〜F24 キーバインド対応、検索可能メニューでのペースト、推論のみのコンパクトなステータス/タイトル項目を追加（#25329, #25400, #25504）。

- **ホスト型 Web/画像ツール**: より多くの code-mode フローで利用可能になり、スタンドアロンの Web 検索を並列実行可能に（#25176, #25702, #25890, #25923）。

- 表示出力前にプロンプトをキャンセルした際、下書き・添付・コラボレーションモードを復元（#25316）。

- 権限要求・承認が環境アイデンティティを保持し、管理された MITM プロキシが子コマンドへ読み取り可能な CA バンドルをエクスポート（#25850, #25858, #25862, #22668）。

### 開発者への影響

`codex plugin list --json` の機械可読出力は CI/自動化に有用です。マルチエージェント v2 のランタイム選択保持とホスト型 Web ツールの並列実行により、複雑なエージェントワークフローの柔軟性が向上します。

## github/copilot-cli

直近7日間で **1.0.59（2026-06-02）** と **1.0.60（2026-06-05）** がリリースされました。

### 1.0.60 の主な変更点

- **最大推論努力レベル（Anthropic モデル）**: Anthropic モデル向けに最大推論努力レベルを追加し、全プランですべての努力レベルを利用可能に。
- **`builtInAgents.rubberDuckAutoInvoke` 設定**: ラバーダック（rubber duck）エージェントの自動起動を制御（デフォルト無効）。
- **`ctrl+s` の再割り当て**: 現在のプロンプトを stash / pop する操作に変更（Claude Code との挙動統一）。スラッシュコマンドピッカーは `/` 入力で引き続き利用可能。
- **`/context` の強化**: カスタムインストラクションをシステムプロンプトと分離し、サーバーごとの MCP ツールトークンコストを `/mcp` とクロスリファレンス。
- **Vim 風ナビゲーション**: `/diff` ビューに g, G, Ctrl+D, Ctrl+U を追加。
- **PR からのワークツリー作成**: PR 一覧画面から PR 用 git ワークツリーを直接作成。Rewind ピッカーは各チェックポイントの作業ツリー diff 統計（+追加 −削除）を表示。
- **`-r`（`--resume` のショートハンド）追加**、`billing` ヘルプトピック追加。
- **セキュリティ修正**: `web_fetch` がループバック・プライベート・クラウドメタデータアドレスをブロックし、リダイレクトを暗黙的に追従しないように。
- カスタムエージェントインストラクションが毎ターン重複しないようになり、コンテキストウィンドウ使用量を削減。

### 1.0.59 の主な変更点

- **`/voice` コマンド**: ローカルの音声認識（speech-to-text）モデルでプロンプトを口述入力。

### 開発者への影響

`web_fetch` の SSRF 対策（プライベート/メタデータアドレスのブロック、リダイレクト非追従）はセキュリティ上重要です。カスタムインストラクションの重複解消によりコンテキスト効率が改善されます。

- リリース (1.0.60): https://github.com/github/copilot-cli/releases/tag/v1.0.60
- リリース (1.0.59): https://github.com/github/copilot-cli/releases/tag/v1.0.59
