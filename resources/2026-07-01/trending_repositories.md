## GitHub Trending: AI駆動開発注目リポジトリ

### 📐 DESIGN.md: Googleが提供するコーディングエージェント向けのデザインフォーマット仕様

[google-labs-code/design.md](https://github.com/google-labs-code/design.md)

Google Labsが公式で提供する、コーディングエージェントにデザインシステムを理解させるためのフォーマット仕様。DESIGN.mdファイルを作成するだけで、AIアシスタントがそのアプリケーションの視覚的アイデンティティを永続的かつ構造的に理解できるようになる。

**仕様の構造**: YAMLフロントマター（機械可読なデザイントークン）とMarkdown本文（人間可読なデザインの理屈）の2層構造。トークンがエージェントに正確な値を与え、散文が「なぜその値なのか」「どう適用するか」を伝える。

**含まれるトークン**:
- `colors`: カラーパレット（16進数、RGB、oklch()など）
- `typography`: フォントファミリー、サイズ、ウェイト、行間、字間
- `rounded`: 角丸サイズ（4px、8pxなど）
- `spacing`: スペーススケール（8px、16pxなど）
- `components`: ボタン、カードなどのコンポーネント別トークン

**CLIツール**: `npm install @google/design.md` でインストール。3つのコマンドを提供:

- `lint`: DESIGN.mdの構造検証、コントラスト比チェック、トークン参照の整合性確認
- `diff`: 2つのバージョン間のトークン変更検出
- `export`: Tailwind v3/v4、W3C DTCGなどのフォーマットに変換

**バリデーション例**:
```bash
npx @google/design.md lint DESIGN.md
# WCAG AAコントラスト比、壊れたトークン参照、孤立トークンなどを検出
```

**なぜ重要か**: FigmaファイルやPDFのデザインガイドはエージェントが直接読めない。DESIGN.mdは「コードと同じリポジトリに置ける、機械可読なデザインソース」を提供する。エージェントがこのファイルを読めば、新しいコンポーネントを作るときも正しい色、タイポグラフィ、スペースを自動的に適用できる。

**エージェントとの統合**: Claude Code、GitHub Copilot、Cursorなど主要なコーディングエージェントで動作。エージェントがコードを生成する際、DESIGN.mdのトークンを参照して一貫性のあるUIを出力できるようになる。

**ステータス**: 現在alphaバージョン。仕様とCLIは活発に開発中で、変更の可能性あり。MITライセンス。

**主なユースケース**:
- デザインシステムのドキュメント化とエージェント共有
- UI生成の一貫性担保
- チーム間のデザイン知識の伝達
- アクセシビリティ（WCAGコントラスト）の自動検証

### 🤖 Agent-Native: エージェントとUIが深く統合されたアプリを構築するフレームワーク

[BuilderIO/agent-native](https://github.com/BuilderIO/agent-native)

BuilderIOが提供する、AIエージェントを「チャットボットとして隣に置く」のではなく「アプリ内部で動作する第一級市民」とするためのフレームワーク。アクション、状態、ID、ツール、スキル、ジョブ、可観測性、UIサーフェスがすべて一体となって動作する。

**核心概念「アクション」**: 1回定義するだけで、UI、エージェント、HTTP API、MCP、A2A（エージェント間通信）、CLIのすべてから呼び出せる。

```typescript
// 1つのアクション定義
export default defineAction({
  schema: z.object({
    emailId: z.string(),
    body: z.string(),
  }),
  run: async ({ emailId, body }) => {
    await db.insert(replies).values({ emailId, body });
  },
});
// UIボタン、エージェントのツール、APIエンドポイント、CLIコマンドすべてで同じ実装を使う
```

**エージェントとUIの完全統合**:
- 1つのデータベース、1つの状態。どちら側からの変更も即座に同期
- 人間とエージェントが同じドキュメントを同時編集（エージェントは第一級ピア）
- エージェントはユーザーが見ているものを認識。テキストを選択してCmd+Iでコンテキスト-awareな指示
- エージェントが他のエージェントを呼び出して協調（タグベースのA2A）
- エージェントが自ら機能追加、バグ修正、UI改善を行える

**テンプレート**: 完全なSaaSアプリとしてクローン可能:

- **Clips**: Loom + Jam相当。画面録画＋自動トランスクリプト＋ブラウザデバッグログキャプチャ。エージェントがトランスクリプトとタイムスタンプ付きフレームを見てバグを修正
- **Plans**: コーディングエージェント用のビジュアルプランモード。`/visual-plan`で構造化されたレビュー可能なプランを生成、`/visual-recap`でPR/diffをハイレベルなビジュアル recapに変換
- **Design**: HTMLプロトタイプ生成、バリアント比較、コントロール微調整、出力
- **Content**: Obsidian相当のMDXエディタ。ローカルMarkdown/MDX編集、リッチなカスタムブロック生成、エージェントによる下書き/書き直し/公開
- **Slides**: Google Slides/Pitch相当。Reactベースのプレゼンテーションをプロンプトまたはポイント＆クリックで生成編集
- **Analytics**: Amplitude/Mixpanel相当。分析データソース接続、プロンプトからリアルタイムチャート生成、再利用可能ダッシュボード構築

**バックエンドに依存しない**: Drizzleがサポートする任意のSQLデータベース、Nitro互換の任意のホスティングプロバイダを使用可能。自分のDB、ホスト、モデルスタック、アプリコードを持ち込める。

**スキルで試す**: アプリ全体をscaffoldしなくても、既存のエージェントに機能追加:

```bash
npx @agent-native/core@latest skills add visual-plan
```

Claude Code、Codex、Cursor、Pi、OpenCode、GitHub Copilot/VS Codeなどに`/visual-plan`と`/visual-recap`スラッシュコマンドが追加される。

**クイックスタート**:

```bash
npx @agent-native/core@latest create my-app
cd my-app
pnpm install
pnpm dev
```

`create`はテンプレート選択（フルテンプレート、チャットのみ、ヘッドレス）を prompts でガイド。

**既存アプローチとの比較**:

| 特徴 | SaaSツール | 生のAIエージェント | 社内ツール | Agent-Native |
|------|-----------|-------------------|-----------|---------------|
| UI | 洗練されてるが硬い | なし | 品質まちまち | 完全なUI、forkして即座にカスタム |
| AI | 後付け | 強力 | 浅い接続 | エージェントファースト、統合 |
| カスタマイズ | 不可 | 指示とスキル | フルだがメンテ重い | エージェントがアプリを修正 |
| 所有権 | レンタル | ある程度自分のもの | コードを所有 | コードを所有 |

**主な特徴**:
- **アクション**: 1回の定義で6つの場所（UI、エージェント、API、MCP、A2A、CLI）で使用
- **エージェントランタイム**: チャット、ツール、スキル、メモリ、ジョブ、可観測性、ハンドオフが一緒に出荷
- **ID・認証**: 組み込みのユーザー管理
- **SQL-backed状態**: Drizzle ORMを使用した永続化
- **マルチテナンシー対応**: チーム、組織、権限管理
- **Jobs**: 定期実行、キューベースのバックグラウンド処理
- **可観測性**: 組み込みのロギング、メトリクス、トレース

**なぜ重要か**: 既存の「エージェントを使う」アプローチは、エージェントをUIの横に置く補助ツールとして扱っている。Agent-Nativeはエージェントをアプリのコア構成要素として扱う — アクション定義そのものがエージェントのツールになり、UIもエージェントも同じアクションを呼び出し、同じDBを見る。これにより「エージェントが機能を追加できる」「バグを直せる」「UIを改善できる」という自己進化するアプリが実現可能になる。

**技術スタック**: TypeScript（72.2%）、MDX（24.2%）、JavaScript（2.1%）。Reactベース。MITライセンス。
