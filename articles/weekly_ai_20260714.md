今週は、フロンティアモデルの供給とコーディングエージェントの境界整備が同時に進んだ、密度の高い一週間でした。
OpenAI が GPT-5.6 ファミリー（Sol/Terra/Luna）を API と ChatGPT で公開し、Anthropic は輸出規制解除を受けて Claude Fable 5 を再展開しました。二大フロンティアが揃って開発者の手元に戻った一週間です。
一方でツール側は「安全に、安く、使い倒す」方向に揃っています。GitHub Copilot CLI は GPT-5.6 対応と同時に `--sandbox` と `preToolUse` 拒否でエージェントの安全境界を強化し、Claude Code は Bedrock/Vertex/Foundry で Opus 4.8 をデフォルト化して `/doctor` による CLAUDE.md 健全化を入れました。
海外コミュニティでも「GitHub の AI エージェントを騙してプライベートリポジトリを漏洩させる」実証や「プロンプト読込前に 33k トークン消費」という計測が話題で、性能ではなく「どう安全に走らせるか」が主戦場になっています。

## 🚀 リリース情報

### Claude Code v2.1.207 / v2.1.206

Auto mode が `CLAUDE_CODE_ENABLE_AUTO_MODE` のオプトインなしで Bedrock・Vertex AI・Foundry で使えるようになり、無効化は設定の `disableAutoMode` に統一されました。
マネージド基盤で Auto mode の前提が揃ったことに加え、Bedrock・Vertex・Claude Platform on AWS が Claude Opus 4.8 をデフォルト化しました。環境ごとのモデル乖離が減り、移行時の挙動差を気にせず Auto mode を試せる状態が整っています。

v2.1.206 では運用に直結する改善が並びました。`/doctor` がチェックイン済みの `CLAUDE.md` の肥大化を検知し、コードから導出できる内容の削減を提案します。
`/commit-push-pr` は `remote.pushDefault`（または唯一のリモート）への `git push` を自動許可するようになり、PR までの一連操作の手戻りが減りました。
また MCP サーバーの per-server `request_timeout_ms` が `--mcp-config` や `.mcp.json` で無視される問題が修正され、長時間の MCP ツール呼び出しが 60 秒のデフォルトで切られる事故を防げます。

- リポジトリ: https://github.com/anthropics/claude-code
- CHANGELOG: https://github.com/anthropics/claude-code/blob/main/CHANGELOG.md

### GitHub Copilot CLI v1.0.70

GPT-5.6 モデルサポートの追加が目玉ですが、それ以上にエージェントの安全境界が厚くなりました。
`--sandbox` / `--no-sandbox` フラグでセッション単位の OS レベルシェルサンドボックスを切り替えられ、保存設定を変えずに `-p` と併用できます。`preToolUse` フックが exit code 2 でツール呼び出しを拒否できるようになり、危険な操作を事前に止める制御点が増えました。

チーム運用向けには `.github/copilot/settings.json` で信頼済みリポジトリがモデル・effort・context tier を固定し、URL/MCP/skill の deny リストを拡張できます。組織で許容する範囲をコードとして定義できるため、導入時のガバナンスに直結します。
`/refine` は荒いプロンプトを明確な指示に書き直すコマンドで、Forge がワークフローパターンを検出してドラフトスキルを作成する流れと合わせると、エージェントへの指示作りが CLI 内で完結します。

- リポジトリ: https://github.com/github/copilot-cli
- リリース: https://github.com/github/copilot-cli/releases/tag/v1.0.70

### Long-Horizon-Terminal-Bench

AI エージェントは短時間の単純タスクを自律実行できるようになりましたが、既存のターミナルベンチマークは数分で終わる問題を最終結果のみで評価するため、中間進捗や部分解が見落とされていました。
本論文は実験再現・ソフトウェアエンジニアリング・マルチモーダル分析・ゲーム・科学計算など 9 カテゴリ 46 タスクからなる長時間ターミナルベンチマークを提案します。各タスクを細粒度のサブタスクに分解し、密な中間報酬と部分点で「最終到達だけでなく途中どれだけ進んだか」を評価する設計です。
15 のフロンティアモデルを評価した結果、最強モデルでも完全達成率 10.9%、平均 4.3% という低い成功率でした。長時間計画・長文脈管理・反復デバッグ能力に大きな改善余地があることが示されています。

- 著者: Zongxia Li, Zhongzhi Li, Yucheng Shi, Ruhan Wang, Junyao Yang, Zhichao Liu, Xiyang Wu, Anhao Li, Yue Yu, Ninghao Liu, Lichao Sun, Haotao Mi, Leowei Liang
- arXiv: https://arxiv.org/abs/2607.08964

### Scalable Visual Pretraining for Language Intelligence

大規模基盤モデルの進歩はテキストコーパスでの事前学習が牽引してきましたが、図表・数式・ページレイアウトなど視覚表現に込められた知識はテキストのみでは忠実に捉えられません。
本論文は「言語モデルはテキスト表現でのみ学習すべき」という前提に異を唱え、視覚事前学習が基盤モデルの知能をスケールさせる学習器であることを示します。テキスト抽出を行わず視覚文書を直接活用する教師なし視覚事前学習パラダイムを体系的に研究し、同一コーパスでも視覚事前学習が一貫してテキストのみの事前学習を上回ることを実証しました。
文書やウェブページが持つ視覚的手がかりを破棄せず取り込む方向は、ドキュメント理解を含む実務タスクに効いてくる見方です。

- 著者: Yiming Zhang, Zhonghan Zhao, Wenwei Zhang, Haiteng Zhao, Tianyang Lin, Yunhua Zhou, Demin Song, Kuikun Liu, Haochen Ye, Haian Huang, Yuzhe Gu, Haijun Lv, Qipeng Guo, Bin Liu, Gaoang Wang, Kai Chen
- arXiv: https://arxiv.org/abs/2607.09657

### Video Generation Models are General-Purpose Vision Learners (GenCeption)

NLP が次トークン予測で汎用基盤モデルに移行したように、コンピュータビジョンにおける汎用モデル実現の触媒は何かという問いに答えます。
本論文は大規模テキスト to 動画生成が CV の強力な事前学習パラダイムであると主張し、動画生成拡散バックボーンを用いたフィードフォワード知覚モデル GenCeption を提案します。テキスト指示で深度・表面法線・カメラ姿勢推定・参照表現セグメンテーション・3D キーポイント予測など多様なタスクを実行し、DepthAnything3 や SAM3 などの専門モデルに匹敵・超越する性能を達成しました。
7〜500 倍少ない学習データで同等性能を実現するデータ効率性と、合成人間動画のみで学習したモデルが実世界映像や未学習カテゴリに汎化する創発的挙動も確認されています。

- 著者: Letian Wang, Chuhan Zhang, Rishabh Kabra, Jasper Uijlings, Steven Waslander, Andrew Zisserman, Joao Carreira, Kaiming He, Misha Andriluka, Eduard Gabriel Bazavan, Andrei Zanfir, Cristian Sminchisescu
- arXiv: https://arxiv.org/abs/2607.09024
