## Claude Code (v2.1.202) — バックグラウンドエージェント安定化とDynamic Workflow設定

- **公開日**: 2026-07-06
- **リポジトリ**: https://github.com/anthropics/claude-code
- **CHANGELOG**: https://github.com/anthropics/claude-code/blob/main/CHANGELOG.md

### 主な変更点

**新機能**
- **Dynamic Workflow サイズ設定**: `/config` に "Dynamic workflow size" 設定（small/medium/large）を追加。Claudeが生成するワークフローのエージェント数をガイドラインとして制御可能（強制キャップではなく目安）
- **OpenTelemetry 属性拡張**: ワークフロー起動エージェントのテレメトリに `workflow.run_id` と `workflow.name` を追加し、OTelデータからワークフロー実行を再構築可能に

**重要なバグ修正（多数）**
- バックグラウンドエージェントのデーモン安定性を大幅改善（Linux自己終了バグ、macOS SSH起動失敗、`daemon.lock` ステイル問題など）
- サブエージェントがレート制限/サーバーエラーでサイレント失敗していた問題を修正し、部分結果を親に返すよう挙動改善
- ストリーミング応答が途中のoverloadedエラーで破棄されていた問題を修正（部分出力を保持）
- `claude agents --plugin-dir` でエージェント/スキルが表示されないバグを修正
- Remote Control（モバイル/Web）からインタラクティブセッションへのコマンド送信エラーを修正
- 音声ディクテーションのマイク失敗時の無限再試行ループを修正
- SSL証明書エラーで無駄に再試行を消費していた問題を、即時失敗＋修正ヒント表示に改善
- tmux 3.4+ でのレンダリングフリッカーを同期出力で解消

### `/review` 挙動変更

`/review <pr>` が高速シングルパスレビューに戻りました。マルチエージェントレビューは `/code-review <level> <pr#>` を使用してください。

## Cline (CLI v3.0.36 / v3.0.37 / v3.0.38) — UI刷新とPlan/Actモード改善

- **公開日**: 2026-07-03 / 2026-07-04 / 2026-07-07（3連続リリース）
- **リポジトリ**: https://github.com/cline/cline
- **最新リリース**: https://github.com/cline/cline/releases/tag/cli-v3.0.38

### v3.0.38 — 新しいブランドパレット（2026-07-07）

**ビジュアル刷新**
- Plan/Actモードのアクセントカラー変更: Act = Blue（`#79b8ff`）、Plan = Amber。ダイアログ、モデルセレクター、設定、オンボーディング、マークダウン、シンタックスハイライト全体に適用
- チャット入力を再デザイン: tinted背景を廃止し、full-width水平線 + 太字アクセントプロンプトグリフに変更。ユーザーメッセージバブルをスリム化
- アシスタントマークダウンのアクセントが、生成時のモード（Plan/Act）で色分けされるように

**機能改善**
- thinking-level ピッカーのデフォルトカーソルが Medium に
- `read_files` が弱いモデルからの不正入力を許容（別配列アイテムとして送られる `start_line`/`end_line` を直前のファイルパスに統合）
- カタログにコンテキストウィンドウ未報告のモデルはデフォルト128Kトークン上限を適用

### v3.0.37 — Plan/Actモード可視性（2026-07-04）

- モード切替がモデルから見えるようになり、セッション途中の切り替えを認識可能に
- Plan/Actモード通知がプロンプトから欠落するバグを修正
- 空セッションでのモード切替による予期しない再起動の競合を修正

### v3.0.36 — Plan→Act切替の即時反映（2026-07-03）

- Planモードの `switch_to_act_mode` ツールがターン終了まで反映されないバグを修正。切替が即座にPlan実行を終了し、承認済みプランでActモードの全ツールセットに移行するように
- Tabモードトグルが完了中ターンと競合して未承認のPlan実行を自動開始する問題を防止

## Visual Studio Code (1.127.0) — Agents window強化、ブラウザツールGA、サンドボックス

- **公開日**: 2026-07-01
- **リポジトリ**: https://github.com/microsoft/vscode
- **リリースノート**: https://github.com/microsoft/vscode/releases/tag/1.127.0
- **詳細**: https://code.visualstudio.com/updates/v1_127

### エージェント・Copilot関連の新機能（AI機能のみ抜粋）

**Agents window（プレビュー）**
- **セッショングループ**: 関連セッションをグループ化して整理可能。グループ単位で新規セッション開始・一括完了マーク可能
- **ドラッグ&ドロップ**: セッションの並べ替え、グループへのドラッグ、Pinnedセクションへのドロップ、複数セッションの同時移動をサポート
- **CI/PRコメントバナー**: PRのCI失敗やレビューコメントをチャット入力欄上部にバナー表示。「Fix Checks」「Address Comments」でエージェントに直接対応可能
- **エディターガターからのフィードバック**: 行をホバーするとgutterに「Add Feedback」グリフが表示され、その行にコメントをドロップ可能

**マルチチャットセッション改善**
- チャットのClose/Reopen/Delete、全チャット横断での進捗と変更の集約、会話のピアチャットへのフォーク機能

**コスト管理**
- **サブエージェントクレジット**: サブエージェントをホバーすると、そのサブエージェントが消費したAIクレジットを表示

**ターミナルコマンドのサンドボックス化（macOS/Linux）**
- エージェントが実行するターミナルコマンドがデフォルトでサンドボックス実行（ネットワーク遮断・ファイルシステム制限）。承認プロンプトを大幅削減。サンドボックス外での実行が必要な場合のみ承認を要求

**統合ブラウザ**
- **ブラウザツールGA**: エージェントがブラウザでページを開き、コンテンツ読み取り、スクリーンショット、クリック/入力/ナビゲーションで自分の作業を検証可能。外部MCPサーバー不要でデフォルト有効
- **パーサイト権限**: カメラ、位置情報、Bluetooth、USBなど、サイトごとに権限を許可/拒否

**`/troubleshoot` 拡張**
- エージェントホストセッション（ローカル・リモート）の診断に対応

**MCPサーバー認証管理**

### Kiro Web（2026-07-01）

- 設定: `Settings > Agent > Sandbox > AWS Credentials`

**Authorize Powers for Third-Party Services**
- Figma、Stripe、Supabaseなどのサードパーティサービス接続Powersが標準認証フローに対応。一度認可すれば以降のタスクでプロンプトなしで利用可能

### Claude Sonnet 5 利用可能（2026-07-01）

- Kiro IDE、CLI、WebでClaude Sonnet 5が利用可能に
- Anthropicの最もアジェンティックなSonnetモデル。Sonnet 4.6からの大幅アップグレードで、推論・ツール使用・コーディングがOpus 4.8に迫る性能（Sonnetクラスの価格）
- 1Mコンテキストウィンドウ、1.3xクレジット倍率（Sonnet 4.6と同等）
- AWS US-East-1（北バージニアニア）およびAWS Europe（フランクフルト）リージョンで、Pro/Pro+/Pro Max/Powerプランのユーザーに段階的ロールアウト（クロスリージョン推論サポート）

### アドオンクレジット（個人プラン）（2026-07-01）

- Pro、Pro+、Pro Max、Powerプランのユーザーが月次割り当て消費後にアドオンクレジットを購入可能

- 12ヶ月有効、未使用クレジットは期限まで月次ロールオーバー
- 月次割り当てを先に消費し、その後アドオンクレジットに移行

## Cursor — Background Agents in Slack

- **公開日**: 直近（日付非公開・最新エントリ）
- **ウェブサイト**: https://cursor.sh
- **チェンジログ**: https://cursor.sh/changelog

### 新機能

**Slack連携バックグラウンドエージェント**
- Slackで `@Cursor` をメンションすることで、Background Agentを直接起動可能
- エージェントがスレッド全体を読んで文脈を把握し、GitHubにPRを作成
- スレッド内で課題調査や質問への回答も可能

**セットアップ**
1. 管理者が Dashboard → Integrations からSlack統合を設定
2. 任意のチャンネルで `@Cursor` + プロンプトを送信
3. `help` コマンドで全コマンド表示、`settings` でデフォルトモデル/リポ/ブランチを設定

**改善（1件）**
- 設定で `Cmd/Ctrl+F` による検索が可能に

**MCP改善（3件）**
- サーバーからの進捗通知、ダイナミックツール登録、Rootsサポート

**パッチ（1.1.1〜1.1.4）**
- 1.1.1: ワークスペースインデックス問題の修正
- 1.1.2: クライアント側パフォーマンス改善
- 1.1.3: MCPパフォーマンス回帰修正、PR検索&インデックス追加、deeplink体験改善、上流マーケットプレースプロバイダー変更オプション
- 1.1.4: Background Agent信頼性向上、Chat UI検索バグ修正

> 注意: Cursorのチェンジログは最新エントリに明示的な日付を掲載していません。最新エントリとして表示されているため今週のリリースと判断しましたが、公開日の厳密な確認はできていません。
