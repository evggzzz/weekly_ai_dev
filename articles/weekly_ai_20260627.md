## 📈 注目のAI開発リポジトリ

### OpenMontage: AIコーディングアシスタントを動画制作スタジオに変える

https://github.com/calesthio/OpenMontage

世界初のオープンソース・エージェント動画制作システム。AI コーディングアシスタントに自然言語で指示するだけで、企画・脚本・アセット生成・編集・最終構成まで自動処理します。12のプロダクションパイプライン、52の制作ツール、500以上のエージェントスキルを備え、参照動画（YouTube/Short/Reel/TikTok）を貼るだけでトランスクリプト・ペーシング・シーン・キーフレーム・スタイルを分析し、差別化されたコンセプトを2〜3個提案。Piper TTS、Archive.org/NASA/Wikimedia Commons の無料映像、Remotion、FFmpeg、WhisperX を組み合わせれば API キーなしでも起動でき、GPU があれば WAN 2.1 / Hunyuan / CogVideo / LTX-Video でローカル動画生成も可能です。実行前見積もり・予算上限・アクションごとの承認しきい値でコストガバナンスも内蔵されています。

### AWS公式のAIエージェントツールキット

https://github.com/aws/agent-toolkit-for-aws

AWS が公式提供する AI コーディングエージェント向けツールキット。MCP サーバー、スキル、プラグインを通じて、Claude Code / Codex / Cursor / Kiro などのエージェントが AWS サービスを構築・デプロイ・管理できるようにします。300以上の AWS サービスを単一認証エンドポイントから操作可能な MCP サーバー、`aws-core` / `aws-agents` / `aws-data-analytics` / `aws-agents-for-devsecops` のプラグイン群、エージェント専用の IAM 条件キー（エージェントアクションとヒューマンアクションを区別）、CloudWatch メトリクスと CloudTrail 監査ログによるエンドツーエンドの可観測性が特徴です。Claude Code では `/plugin install aws-core@claude-plugins-official` で即座に利用開始できます。

### MemSlides: A Hierarchical Memory Driven Agent Framework for Personalized Slide Generation with Multi-turn Local Revision

- **著者**: Ye Jin, Yangyang Xu, Jun Zhu, Yibo Yang
- **arXiv**: https://arxiv.org/abs/2606.17162

プレゼンテーション生成のための階層的メモリフレームワーク。長期記憶（ユーザープロファイル + ツール実行履歴）と短期記憶（セッション制約）を分離し、マルチターン修正時の安定したパーソナライズを実現。スライド単位の局所編集により、再生成による意図しない変更を最小化します。実験では、プロファイル記憶がペルソナ適合性を向上させ、ツール記憶が修正動作の安定性に寄与することを確認しています。

### Qwen-AgentWorld: Language World Models for General Agents

- **著者**: Yuxin Zuo, Zikai Xiao, Li Sheng, Fei Huang, et al. (33名)
- **arXiv**: https://arxiv.org/abs/2606.24597

7ドメインのエージェント環境シミュレーション可能な言語ワールドモデル（Qwen-AgentWorld-35B-A3B / 397B-A17B）を開発。1000万以上の環境相互作用軌跡で訓練し、長い Chain-of-Thought 推論により次状態予測を実現。世界モデルは（i）環境シミュレータとして強化学習トレーニングを支援、（ii）統一エージェント基盤モデルとして下流タスク性能を向上。AgentWorldBench 評価で既存最先端モデルを大きく上回る結果を報告しています。

### Are We Ready For An Agent-Native Memory System?

- **著者**: Wei Zhou, Xuanhe Zhou, Shaokun Han, Hongming Xu, Guoliang Li, Zhiyu Li, Feiyu Xiong, Fan Wu
- **arXiv**: https://arxiv.org/abs/2606.24775

LLM エージェントのメモリシステムをデータ管理観点から体系的に評価。メモリを表現・記憶、抽出、検索・ルーティング、保守の4モジュールに分解し、12システムを5ワークロード（11データセット）で評価。単一アーキテクチャですべてのシナリオで優位ではなく、有効性はワークロードボトルネックとの整合性に依存することを示しました。局所的保守が全体的再編成よりもコスト効率が高いことを明示し、エージェントネイティブメモリシステムへの方向性を提示しています。
