# トレンドリポジトリ

今週の GitHub トレンド（直近7日）から、AI駆動開発に直結するリポジトリを2つ厳選した。

## openai/codex-plugin-cc

- **リポジトリ**: https://github.com/openai/codex-plugin-cc
- **一言で**: Claude Code の中から Codex にコードレビューやタスクを丸投げできる、OpenAI 公式プラグイン。

### 何ができるのか

Claude Code で作業している最中に、Codex のレビュー機能や推論能力をそのまま呼び出せる。スラッシュコマンドで切り替え可能で、たとえば次のような使い方ができる。

- `/codex:review` — 未コミット変更やブランチ差分に対する通常の読み取り専用レビュー。`--background` でバックグラウンド実行。
- `/codex:adversarial-review` — 設計判断を突き崩す「対立レビュー」。認証・データ喪失・競合状態・ロールバックといったリスク領域を重点的に問う。
- `/codex:rescue` — バグ調査やパッチ作成を Codex に委任。`--resume` で前回スレッドを継続、`--model gpt-5.4-mini` のようにモデルと推論努力量を指定できる。
- `/codex:transfer` — Claude Code 側の会話コンテキストを Codex のセッションに持ち出して、そのまま Codex CLI / App で続きを書ける。
- `/codex:status` / `/codex:result` / `/codex:cancel` — バックグラウンドタスクの進捗確認・結果取得・キャンセル。

必要なのは Node.js 18.18 以上と ChatGPT アカウント（Free 含む）または OpenAI API キーのみ。プラグインはローカルの `codex` CLI を呼び出すため、既存の認証・設定（`~/.codex/config.toml`）がそのまま効く。

### なぜ今注目すべきか

これまで Claude ユーザーが Codex を使うにはターミナルを切り替える必要があった。このプラグインは「Claude Code の文脈を保ったまま別モデルの目を借りる」という導線を公式に用意したことで、複数エージェントの協調が OSS の標準機能になりつつあることを示している。

特に `/codex:adversarial-review` は実用的だ。自分が書いたコードを別モデルに「その設計選択で本当に良かったのか？」と問い直させる運用は、レビューの盲点を潰す手段として強力。さらに `/codex:setup --enable-review-gate` を有効にすると、Claude がタスク完了を返す前に Codex が自動で指摘を入れるゲートも組める（ただし使用量を消費しやすいと明記されている）。

26,000 スター超・今週4,300スター増という伸びは、マルチエージェント運用が一部のハッカーだけの話ではなくなっている証拠。Claude Code を主戦場にしている開発者が、Codex の「別の視点」を手軽に取り込める最初の公式導線として要チェックだ。

## ogulcancelik/herdr

- **リポジトリ**: https://github.com/ogulcancelik/herdr
- **公式サイト**: https://herdr.dev
- **一言で**: ターミナル住みのエージェントマルチプレクサ。tmux をコーディングエージェント向けに作り直したもの。

### 何ができるのか

複数の Claude Code / Codex / Copilot CLI / Gemini CLI を一つのターミナルで並走させ、それぞれの状態を一覧できる。売りは三点。

1. **エージェントの状態がひと目で分かる** — 各ペインを 🔴 blocked / 🟡 working / 🔵 done / 🟢 idle に自動分類。フック設定なしで、プロセス名と出力内容から判定する。
2. **本物のターミナルをエージェントごとに割り当て** — フルスクリーン TUI もそのまま描画される。アプリ内の模造ターミナルではない。
3. **デタッチしても死なない** — バックグラウンドサーバーがペインとエージェントを生かし続ける。ノートPCを閉じて、別ターミナルや SSH 経由のスマホから再アタッチ可能。

単一の Rust バイナリ（約10MB）で Linux / macOS 動作（Windows はベータ）。Electron もアカウントもテレメトリもなし。`curl -fsSL https://herdr.dev/install.sh | sh` で入るほか、Homebrew / mise / Nix にも対応。

対応エージェントは Claude Code、Codex、GitHub Copilot CLI、Gemini CLI、Cursor Agent、Devin CLI、Amp、OpenCode、Grok CLI など16種類以上。公式インテグレーションを入れるとネイティブなセッション復元や意味的な状態報告まで追加される。

### なぜ今注目すべきか

エージェントを3〜4本同時に走らせる運用が当たり前になりつつある今、従来の tmux では「どのペインが止まっていて、どれが待機中か」を把握するのが難しい。herdr はそこを「エージェントという単位で状態を可視化する」方向で切り直した。

 tmux 派が一番気になるのは「乗り換えコスト」だが、操作体系は `ctrl+b` プレフィックスなど tmux 互換を踏襲。マウスでペイン分割・ドラッグもできる。さらに `herdr --remote` でリモートVPS上のサーバーにローカルターミナルから接続すると、画像のペーストすらそのまま動く——これは素の `ssh` + `tmux` では壊れる点で、リモート運用の现实な弱点を突いている。

12,800 スター・今週4,300スター増。6月30日には GitHub デイリートレンド1位を記録した。複数エージェントの並走を当たり前にするインフラとして、「ターミナルの中で完結する」という判断基準が明確で、GUI マネージャー派と tmux 派のどちらにも響く設計だ。
