## anthropics/claude-code

- **最新バージョン**: 2.1.176（CHANGELOG 最終更新: 2026-06-12）
- 期間内に 2.1.172〜2.1.176 が更新されました。

### 主な変更点（重要機能を中心に）

#### サブエージェントのネスト実行（2.1.172）

- **サブエージェントが自身のサブエージェントを生成可能に（最大5階層）**: これまで1段だったサブエージェントの委譲が、最大5階層までネスト可能になりました。

サブエージェントは独自のコンテキストウィンドウ・システムプロンプト・ツールアクセス・権限を持つ専門アシスタントで、メインの会話を検索結果やログで圧迫しない用途に有効です。

利用・設定例:

```bash
# サブエージェントの管理（作成・編集・一覧）
/agents
```

サブエージェント定義はプロジェクトレベル（`.claude/agents/`）またはユーザーレベル（`~/.claude/agents/`）にMarkdownファイルとして配置し、`description` フィールドでClaudeが委譲判断を行います。`tools` でツールアクセスを制限したり、`model`（例: Haiku）を指定して高速・低コストなワーカーにルーティングできます。

#### その他の主な変更

- **会話言語でのセッションタイトル生成（2.1.176）**: セッションタイトルが会話の言語で生成されるようになりました（`language` 設定で固定可能）。
- **`footerLinksRegexes` 設定の追加（2.1.176）**: フッター行に正規表現マッチのリンクバッジを表示。ユーザー設定／管理設定から構成可能。
- **`enforceAvailableModels` 管理設定の追加（2.1.175）**: 有効化すると `availableModels` の許可リストが Default モデルにも適用され、ユーザー／プロジェクト設定で管理側のリストを広げられなくなります。
- **Bedrock 認証情報のキャッシュ改善（2.1.176）**: `awsCredentialExport` の認証情報を固定1時間ではなく `Expiration` までキャッシュ。
- **`availableModels` の強制を堅牢化（2.1.176）**: 環境変数経由でブロック対象モデルへ迂回できないよう修正。`/fast` も許可外モデルへの切り替えを拒否。
- **`wheelScrollAccelerationEnabled` 設定の追加（2.1.174）**: フルスクリーン時のマウスホイール加速を無効化可能に。
- フック `if` 条件のパスマッチ修正、tmux/SSH 環境でのクリップボード連携修正、Remote Control 関連の複数修正など、安定性向上が中心。

### 実務的な影響

ネストサブエージェントにより、調査タスクをさらに細分化して委譲できるようになり、大規模リポジトリでのコンテキスト管理が一層柔軟になります。エンタープライズ向けには `enforceAvailableModels` によるモデルガバナンス強化が重要です。

## kiro

- **バージョン**: 2.7.0 CLI
- **公開日**: 2026-06-12

### 主な変更点

1. **`/goal` コマンド**: ゴール駆動ループを起動。エージェントが目標に向かって反復的に作業し、完了を検証してから停止します（デフォルト最大5反復）。単発プロンプトではなく、目標達成までの自律的なループが可能になりました。
2. **Queue Steering（キュー・ステアリング）**: エージェントの作業中に軌道修正メッセージを送信可能。エージェントはツール境界で指示を取り込むため、キャンセル・再起動なしで途中の方向転換ができます。
3. **`/rewind` プレビューの強化**: ターン選択UIに各ターンの要約（ツール呼び出し、変更ファイル、実行コマンド、中間メッセージ）を表示し、分岐点を素早く特定できます。

> 直近では 2026-06-11 に Web 向けの「GitLab Support and Specs in the Browser」、2026-06-10 に「Kiro Pro Max」も公開されています。

### 実務的な影響

`/goal` と Queue Steering により、長時間タスクの自律実行と途中介入の両立が可能になり、エージェント主導の開発フローの制御性が高まります。

### リンク

- 変更履歴: https://kiro.dev/changelog/

## github/copilot-cli

- **バージョン**: 1.0.62
- **公開日**: 2026-06-13

### 主な変更点

- **プラグインが拡張機能を同梱可能に**: プラグインマーケットプレイス経由でインストール可能な拡張を配布できるようになりました。
- **サブエージェントのモデル・推論努力・コンテキストティアを設定可能**: ユーザー設定または `/subagents`（`/agents`）ピッカーから構成できます。
- **diff ビューの強化**: コンテンツ検索・マッチハイライト・n/N ナビゲーション、ファイルツリーサイドバー、インラインコメントエディタを追加。
- **`/app` スラッシュコマンドを追加**: GitHub アプリ（またはブラウザのフォールバック）を開きます。
- **`/every` と `/after` でスラッシュコマンドをスケジュール可能**（例: `/every 1d /chronicle standup`）。
- **Issues / Pull Requests タブで `/` キー検索**（サーバーサイドフィルタリング）、`W` キーでワークツリー作成。
- **YOLO（全許可）インジケータ**をフッターに表示し、カスタム statusLine.command に allow-all 状態を追加。
- Ask/elicitation ダイアログがタイムラインと連動スクロールするよう改善、Kerberos/Negotiate(SPNEGO) による社内フォワードプロキシ自動認証、シェルコマンドの軽量プロセス起動化など多数の改善・修正。

### UI / ワークフローへの影響

サブエージェントのモデル・推論努力をピッカーから細かく制御でき、diff ビューの検索・コメント機能でレビュー作業が CLI 内で完結しやすくなりました。`/every`・`/after` のスラッシュコマンドスケジューリングは定常タスクの自動化に有用です。

- リポジトリ: https://github.com/github/copilot-cli
- リリース: https://github.com/github/copilot-cli/releases/tag/v1.0.62
